JavaScript is a Single Threaded, Object Oriented, High Level Programming Language, at the same time, also non-blocking, asynchronous and concurrent.JavaScript is an interpreted language, for that it needs an interpreter(known as engines) which converts the JS code to a machine code. Most modern web browsers support JavaScript and have a JavaScript engine that runs JavaScript.The web browser loads a web page, parses the HTML, and creates what is known as a Document Object Model (DOM) from the contents. The HTML and CSS are put together by the DOM to create the web page first. Once this is done, the JavaScript is executed in the order the code is written. Whenever we run a JavaScript program inside a web browser, JavaScript code is received by the browser's engine and the engine runs the source code to obtain the output.This results in the DOM being updated by JavaScript code and rendered by the browser.

The event loop is the secret behind JavaScriptâ€™s asynchronous programming. Event loop is a process in which the browser keeps checking both the callback queue and the call stack. JavaScript is a single threaded programming language which implies that it can run a single piece of code at a time. But Web APIs run in separate threads. So when interacting with the runtime, they must inject themselves into the stack within the single thread of the runtime. As a result there's an endless loop, where the JavaScript engine waits for tasks, executes them and then sleeps, waiting for more tasks. In this case the event loop has one simple job, which is to look at the call stack and callback queue. The call stack is a stack of functions that are currently being executed. The callback queue is a queue of functions that are waiting to be executed. Event loop constantly checks whether or not the call stack is empty. If it is empty, new functions are added from the calback queue. If it is not, then the current function call is processed.

Both local storage and session storage are almost identical, work primarily on data storage & have the same API but local storage stores data with no expiration date until the user manually clears the browser cache or until your web app clears the data. On the other hand session storage stores the data only until the window or tab is closed. Session storage is a popular choice when it comes to storing data on a browser and if we want to cache some application data in the browser for later usage local storage will be the preferable choice.
